# 5月版Adobe Experience Manager指南as a Cloud Service

## 升级到5月版

升级您当前的Adobe Experience Manager指南as a Cloud Service(以后称为 *AEM指南as a Cloud Service*)设置：
1. 查看Cloud Services的Git代码，并切换到在与要升级的环境对应的Cloud Services管道中配置的分支。
2. 更新 `<dox.version>` 属性 `/dox/dox.installer/pom.xml` 文件，以将Cloud ServicesGit代码复制到2022.5.144。
3. 提交更改并运行Cloud Services管道以升级到5月版AEM指南as a Cloud Service。

## 新增功能和增强功能

AEM指南as a Cloud Service在5月版本中提供了许多增强功能和新增功能：

### 增强的Web编辑器

* **根据自定义模板创建映射**

现在，您可以获得创建自定义地图模板的强大功能。 您可以使用它们创建DITA映射，以及映射模板中引用的主题模板和映射模板。

![dita模板](assets/dita-templates.png)

您还可以从自定义映射模板中引用其他映射模板和主题模板。 引用的映射模板可以指各种映射模板、主题模板、主题、映射、图像、视频和其他资产。

![创建dita模板](assets/create-dita-template.png)

自定义的映射模板可帮助您非常轻松地复制映射模板和整个引用的文件夹结构。 这些自定义模板对于创建和重新创建具有递归结构和引用的多个映射特别有用。

* 的 **插入关键词** 功能已得到改进。 现在，您可以更轻松地找到要插入的关键词，因为关键词按字母顺序列出。 您还可以通过在搜索框中键入搜索字符串来搜索关键词。

![插入关键词](assets/insert-keyword.png)

* 现在，“存储库视图”文件会批量加载。 每次加载75个文件。 此批量加载非常有效，与加载文件夹中现有的所有文件相比，您可以更快地访问文件。

![加载更多文件](assets/load-more-files.png)

* 您可以在所有XML编辑器屏幕（包括但不限于“预览”和“创作”视图）中渲染包含嵌入数据或链接的SVG图像。

* 默认的XSD/DTD可更新为最新版本

### 改进了翻译流程

* **能够创建范围翻译项目**
如果您只需要为要翻译的项目创建范围，则可以选择 
**创建新的范围翻译项目**. 这不会发送要翻译的副本，并且文件的原始翻译状态会得到维护。

![范围翻译项目](assets/scoping-translation-project.png)

* 如果拒绝翻译作业中一个或多个主题的翻译，则所有被拒绝主题的“进行中”翻译状态将还原为其原始状态。

* 的 **语言** 列表显示语言文件夹及其语言代码。 例如，法语(fr)和德语(de)。

* 翻译功能现在还支持包含国家/地区和语言的语言代码。 例如， `fr-fr`, `en-us`.

![翻译语言](assets/translation-languages.png)

* 加载语言文件夹以外的DITA映射时，后端不会记录任何异常。

有关翻译的更多详细信息，请参阅 *从Web编辑器翻译文档* 部分中的“使用Adobe Experience Manager指南”as a Cloud Service。


### 增强的发布功能

* 您还可以访问 **发布功能板** 从“输出”选项卡中生成输出。 发布功能板中提供了所有活动发布任务的列表。

![已排队输出](assets/queued-output.png)

* 在映射仪表板中，您可以选择多个DITAVAL文件以生成条件化内容。 您可以通过添加或删除文件来维护文件顺序。 您还可以将鼠标悬停在文件名上，以查看存储文件的AEM存储库中的路径。

* **已弃用功能**
AEM as a Cloud Service不再支持为FrameMaker文档生成DITA输出格式。 此DITA选项也已从“映射”功能板的“输出预设”中删除。

### 改进了基于文章的发布

通过XML编辑器，可以在发布到Salesforce配置文件时，将多个产品类别映射到文章。

### 其他功能增强

* 预览模式还支持 `deliveryTarget` DITA中的条件处理属性。 它可用作下拉过滤器中的选项，以及 **受众**, **平台**, **产品**, prop， **其他prop**.
* 已提供选项，用于在Ox中的AEM服务器与本地系统之间强制同步。
